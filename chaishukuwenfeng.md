# chaishukuwenfeng 分支说明

## 分支目标

本分支用于修复拆书库文风启用后的生成链路：

1. 拆书库中任意一本小说点击“启用文风”后，AI 会话与正文生成应优先读取当前启用的文风。
2. 同一项目同一时间只使用一种启用文风，以当前 `enabledStyleId` 指向的文风为准。
3. 删除角色 Skill 卡片中的“绑定到小说人物”按钮入口。

## 修改边界

只修改拆书库文风注入、AI 会话上下文、深度生成正文试写和角色 Skill 面板入口相关代码。
不删除底层角色灵魂绑定数据结构和其他页面的绑定能力，避免影响现有角色灵魂上下文匹配逻辑。

## 验证要求

1. 相关单元测试必须先失败后通过。
2. 运行拆书库文风、深度生成和角色面板相关测试。
3. 运行 `npm.cmd run typecheck`。
4. 运行 `npm.cmd run build`。
5. 运行 `npm.cmd run build:portable`。
