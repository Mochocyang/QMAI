## 20260615-07:40 - 根因修复：识别失败 `(0, t.createHash) is not a function`

### 🐛 真实根因
`character-recognition-engine.ts` 第 1 行：
```ts
import { createHash } from "crypto"
```
`crypto` 是 **Node.js 内置模块**。Vite 把它打到 Tauri 渲染层后，浏览器里 `t.createHash` 是 `undefined`，调用即抛错。

> 这是个经典坑：vitest 用 jsdom + Node API 时**测试能过**，但浏览器运行时直接挂。

### 🔧 修复
把 `stableCharacterId` 改成 **纯 JS 双 32-bit FNV-1a 拼接**（无任何依赖）：
- 第一段 hash 用字符 charCode
- 第二段 hash 混入 charCode + 位置偏移降低冲突
- 输出 16 hex 字符 → `slice(0, 12)` 与原 SHA-256 截断后行为一致
- 跨 Node.js / 浏览器 / Tauri 都能跑

### 📂 改动文件
- 修改（1 个）：`src/lib/novel/book-analysis/character-recognition-engine.ts` — 移除 `import { createHash } from "crypto"`，实现 12 行纯 JS 哈希
