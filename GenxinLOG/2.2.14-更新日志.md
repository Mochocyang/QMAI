# QMAI 2.2.14 更新日志

发布日期：2026-06-13

## 修复内容

### 1. 修复 Issue #10：Frontmatter 写回失败
- **问题**：AI 修改章节时报错"返回内容缺少 frontmatter，已停止写回"
- **修复**：现在自动沿用原章节 frontmatter，并容错代码围栏与缺失标题

### 2. 修复 Issue #9："继续生成下一章"重复生成第一章
- **问题**：提示词中顺带出现的"开篇/第一章"字样会把目标章节劫持为第1章
- **修复**：
  - 提示词中的"开篇/第一章"不再劫持目标章节
  - 本会话记住刚生成、尚未保存的章节号
  - "继续生成下一章"能正确推进到下一章

### 3. 修复 Issue #6：大纲细化生成逻辑矛盾
- **问题**：大纲细化提示"当前项目还没有可用大纲"，但界面显示大纲文件列表
- **修复**：
  - 统一按目录是否已有 .md 文件判断
  - hasOutlineForRefinement 从基于搜索改为直接文件系统检查

### 4. 修复 Issue #8：缺少单章字数控制
- **问题**：无法控制单章生成的目标字数
- **修复**：
  - 新增"单章目标字数"设置（在小说设置中）
  - 章节生成、扩写阈值和"继续生成下一章"提示词都按设置目标执行
  - 支持自定义字数范围（2000-6000字）

### 5. 增强去AI味规则的中文小说适配
- **优化**：
  - 明确保留角色声线、对白毛边、叙事节奏和必要停顿
  - 不要按非虚构文章规则硬删副词或压缩到固定字数
  - 小说中的"似乎""仿佛""缓缓"等词在特定语境下是风格，不是AI味
  - 情感描写可以有一定修饰，不必强制精简到极致
