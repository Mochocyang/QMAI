五、实施建议

第一阶段：修复致命问题（预估 4.5h）

目标：让两阶段流程真正可运行，推荐点击能触发生成，显示内容干净。
  1. F1 + F3 + Q1：通过 props 将 onGenerateSection 和 onSendMessage 传入 OutlineAssistantMessage；推荐选项和下一步推荐点击时调用对应回调；needs_input 用户选择后直接调用 handleSend(buildGenerationPrompt(...), [], { intentPhase: "generation" }) 跳过意图分析
  2. U1：在 displayContent 生成逻辑中移除 intent_clarity 和 next_step 标记块，复用 extractBodyContent 的思路或新增 stripStructuredMarkers 工具函数

第二阶段：补全交互细节（预估 6h）

目标：4类选项完整可用，阶段信息可读，用户体验可感知。
  1. F2：实现"自定义"选项，将 inputRef 传入或添加临时输入框
  2. U2：在 buildOutlineStages 中从 agentToolCalls / intentClarityResult 提取摘要信息，填充 summary 字段
  3. U3 + U4：自动注入前显示过渡提示；推荐选项上方显示分析摘要
  4. M1（部分）：将推荐选项和下一步推荐提取为独立组件

第三阶段：架构完善与质量提升（预估 9h）

目标：outputMode 真正生效，状态机驱动流程，测试覆盖完整。
  1. F4：在 buildGenerationPrompt 和保存流程中使用 outputMode
  2. F5：面板引入状态机驱动
  3. M3：添加集成测试
  4. Q2 + Q4 + M2：闭包修复、推荐过滤优化、prompt 逻辑集中

第四阶段：性能与兼容性打磨（预估 4.5h）

  1. 性能优化：useMemo / React.memo / indexOf
  2. 兼容性：旧数据加载验证、弱模型降级
  3. 边界场景：超长内容、网络中断、模型切换等

六、风险提示

  1. AI 输出格式遵从度不可控：intent_clarity 和 next_step 依赖 AI 严格按格式输出。如模型能力不足或指令被忽略，整个两阶段流程将失效。建议增加 fallback 机制：解析失败时回退到原有直接生成行为。
  2. 意图分析额外一轮 API 调用：clear 情况下先做意图分析再自动注入生成，等于多一轮 API 请求。对于意图已经很明确的场景，这增加了延迟。建议在 buildIntentAnalysisPrompt 中加入"如已有足够信息，尽量输出 clear"的引导。
  3. 闭包过期风险：lastIntentTitle/lastIntentHint 在 handleSend 的 useCallback 闭包中使用，如果用户在意图分析期间切换会话，可能拿到过期值。建议改用 useRef。
